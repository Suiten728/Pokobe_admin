# Security Policy  
Version: 1.0  
Release Date: 2025-12-07  

このプロジェクトの安全性を維持するための基本的な方針です。
本ポリシーは、[LICENSE](https://github.com/Suiten728/Pokobe_admin/blob/main/LICENSE) と併せて適用されます。

## 🔒 1. サポート対象のバージョン

以下のバージョンは継続的にセキュリティサポート対象です：

- 最新の main ブランチ
- 最新リリース版（X.Y 系の最新タグ）

古いバージョン、改造バージョン、フォーク版はサポート対象外です。

## 🛡 2. 脆弱性の報告方法

もし脆弱性（セキュリティ上の問題）を発見した場合は、  
**公開 Issue ではなく、非公開の方法** で連絡してください。

### 報告手段：
- GitHub の **Security → Report a vulnerability**
- または、（作者が指定している場合）DM・メールなどの非公開チャンネル

報告の際は、可能であれば以下の情報を含めてください：

- 問題の内容
- 再現手順
- 影響範囲（Bot が止まる / 情報漏えいの可能性 など）
- 使用しているバージョン

## 🚫 3. 禁止される行為（テスト時含む）

プロジェクトの安全性を守るため、以下の行為は禁止されています：

- Bot の悪用を目的としたテスト（荒らし・迷惑行為など）
- 他者のサーバーやユーザーに影響を与えるテスト
- 違法行為に関連する使用、解析、改造
- デベロッパーツールや API を不正に利用する行為

※ セキュリティ検証は、**必ず自分の環境でのみ** 行ってください。

## 🧩 4. 責任範囲（重要）

MIT ベースのライセンスに基づき、  
**本ソフトウェアによって生じたいかなる損害についても作者（Suiten）は責任を負いません。**

これには以下が含まれます：

- サーバーの破損・停止
- データ消失
- Discord アカウント・サーバーの制限 / Ban
- 法律上のトラブル

利用者自身の責任において、最新バージョンを安全に運用してください。

## 🛠 5. セキュリティ修正の流れ

脆弱性が報告された場合：

1. 内容を確認し、再現テストを実施  
2. 必要に応じて非公開ブランチで修正  
3. 修正が完了したらパッチ版をリリース  
4. SECURITY.md またはリリースノートでお知らせ  

報告者の名前は、希望がない限り公開しません。

## 📅 6. 改定

本ポリシーは必要に応じて更新されます。</br>
更新された場合は、Version と Release Date が更新されます。
